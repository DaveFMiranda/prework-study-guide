HTML

  • The head element contains information about the webpage.
  • The title element tdefines the title of the page, important for SEO
  • "style" defines CSS styles associated with the page
  • "base" defines the base URL for the page
  • "link" connects an external resource
  • "meta" defines metadata such as character set, description, keywords, author, viewport
  • "script" embeds client-side scripts in the HTML doc
  • The body element represents the visible content shown to the user.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Steps for completing the Git flow:
  • 1. stage changes. In command line: git add -A
  • 2. Add commit message: git commit -m""
  • 3. Pull local base branch to local feature branch: git pull origin main
  • 4. Push changes to remote repository: git push origin feature/BRANCH NAME
  • 5. In GitHub: navigate to feature branch, confirm changes, go to Pull Requests, open new pull request (main as base branch, feature as comparison branch), create pull request, merge pull request
  • 6. Delete branch and close issue
  • 7. To update local main branch: git checkout main, THEN git pull origin main
